# Nested Replies with Flat Display and @Mentions — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Allow comments to be replied to at any depth, displayed flat (1 indent level) under the root comment, with `@username` mentions for context.

**Architecture:** Remove the 1-level nesting restriction in `postComment`. Restructure `getComments` to walk up `parentId` chains to find root ancestors, grouping all descendants flat under the root. Add `replyingTo` metadata to replies. Frontend shows `@username` prefix and allows replying to any comment.

**Tech Stack:** Convex (backend), React + Tailwind (frontend), vitest + convex-test (tests)

**Step 3: Rewrite the grouping logic in `getComments`**

Replace lines 47-66 of `convex/comments.ts` with:

```ts
    // Build lookup map for walking parent chains
    const byId = new Map(withScores.map((c) => [c._id, c]));

    // Find root ancestor by walking up parentId chain
    function findRoot(commentId: string): string {
      const c = byId.get(commentId);
      if (!c || !c.parentId) return commentId;
      return findRoot(c.parentId);
    }

    const topLevel = withScores
      .filter((c) => !c.parentId)
      .sort((a, b) => b.score - a.score);

    // Group all non-root comments under their root ancestor
    const repliesByRoot = new Map<string, typeof withScores>();
    for (const c of withScores) {
      if (c.parentId) {
        const rootId = findRoot(c._id);
        if (!repliesByRoot.has(rootId)) repliesByRoot.set(rootId, []);
        repliesByRoot.get(rootId)!.push(c);
      }
    }
    // Sort replies chronologically
    for (const replies of repliesByRoot.values()) {
      replies.sort((a, b) => a.createdAt - b.createdAt);
    }

    return topLevel.map((c) => ({
      ...c,
      replies: (repliesByRoot.get(c._id) ?? []).map((r) => {
        // Add replyingTo when direct parent is not the root
        const directParent = r.parentId ? byId.get(r.parentId) : undefined;
        const isDirectChildOfRoot = r.parentId === c._id;
        return {
          ...r,
          replyingTo:
            !isDirectChildOfRoot && directParent
              ? { username: directParent.username, commentId: directParent._id }
              : undefined,
        };
      }),
    }));
```

### Task 3: Backend — Update `getLatestComments` for deep replies

**Files:**
- Modify: `convex/comments.ts:142-195` (`getLatestComments` handler)

**Step 1: Update the query**

The current `getLatestComments` uses `by_parent` index which only finds direct children. For deep threading, we need to find ALL descendants. Since `getLatestComments` fetches the 50 most recent comments and we don't need full threading there, the simplest approach is: for each comment that is itself a reply (has parentId), walk up to the root and fetch all siblings from `by_listing` filtered in-memory. However, the latest feed only shows a preview — the simplest correct change is to just fetch direct replies using `by_parent` as before (this still works for the feed preview).

The only needed fix: when a deeply-nested comment appears in the latest 50, it should show as a standalone item (it already does — it has `parentId` so `replies` is `[]`). No change needed for `getLatestComments` — the existing behavior is correct for the feed use case.

**Skip this task — no changes needed.**
