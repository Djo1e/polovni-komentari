# Polovni Komentari ‚Äî Project Context

## What is this?

Polovni Komentari is a Chrome extension that adds community comments to car listing pages on [polovniautomobili.com](https://www.polovniautomobili.com) ‚Äî Serbia's largest used car marketplace. Users can post comments on any listing, reply to others, and upvote/downvote comments. Think of it as a Reddit-style comment layer injected on top of an existing website.

The project is an **independent, unaffiliated** project ‚Äî not connected to polovniautomobili.com or Polovni automobili doo in any way.

**Chrome Web Store:** https://chromewebstore.google.com/detail/polovni-komentari/hjaehnglllmhjbbflknfpjofpnocmnkl

### Architecture

The extension is a **content script only** ‚Äî no popup, no background/service worker, no options page. When a user visits `https://www.polovniautomobili.com/*`, the content script injects a React app into the page via **Shadow DOM** (to isolate styles from the host page).

```
User visits polovniautomobili.com/auto-oglasi/12345/...
  ‚Üí Content script activates
  ‚Üí Extracts listingId from URL (/auto-oglasi/(\d+)/)
  ‚Üí Extracts listing info from DOM (title, price, image)
  ‚Üí Creates Shadow DOM host (#pa-comments-host)
  ‚Üí Renders React app inside shadow root
  ‚Üí Connects to Convex backend for comments/votes
```

### User Identity

There is **no authentication**. Users are identified by:

- **Anonymous ID**: `crypto.randomUUID()` generated on first use, stored in `localStorage` as `paCommentsAnonymousId`
- **Username**: Auto-generated from car brand + 3-digit number (e.g., "Golf234", "Punto891") ‚Äî 25 car brands in the pool. Users can set a custom username, which flips the `isAutoGenerated` flag to `false`

### UI Layout

A fixed-position drawer on the right side of the screen:

- **Toggle button**: Orange circle with comment count badge, positioned on the left edge of the drawer
- **Two tabs** (when on a listing page):
  - "Ovaj oglas" (This Listing) ‚Äî comments for the current listing, sorted by score
  - "Najnoviji" (Latest) ‚Äî global feed of the 50 most recent comments across all listings
- **One tab** (when not on a listing page): just "Najnoviji"
- Drawer state (open/closed) persists in `localStorage`

### Comment System

- Comments are sorted by **score** (upvotes minus downvotes), not chronologically
- **Threading**: One level of replies only (comment ‚Üí reply, no reply-to-reply)
- **Voting**: Up/down toggle ‚Äî click same direction to remove, click opposite to switch. Users cannot vote on their own comments
- **Character limit**: 1000 characters per comment
- **Score calculation**: Computed at query time by summing votes, not stored denormalized

### Convex Backend

**Deployment URL:** `https://notable-goose-213.convex.cloud`

#### Schema (3 tables)

**comments**
| Field | Type | Notes |
|-------|------|-------|
| listingId | string | Links to listing |
| authorId | string | Anonymous UUID |
| text | string | Max 1000 chars |
| createdAt | number | `Date.now()` timestamp |
| parentId | optional id | Self-reference for replies |
| username | string | Display name |
| isAutoGenerated | boolean | Auto-generated username flag |

Indexes: `by_listing`, `by_parent`, `by_createdAt`

**votes**
| Field | Type | Notes |
|-------|------|-------|
| commentId | id | Reference to comment |
| voterId | string | Anonymous UUID |
| direction | "up" \| "down" | Vote direction |

Indexes: `by_comment`, `by_comment_voter` (composite)

**listings**
| Field | Type | Notes |
|-------|------|-------|
| listingId | string | External listing ID |
| title | string | Listing title |
| price | string | Price text |
| imageUrl | string | Thumbnail |
| url | string | Full URL |

Index: `by_listingId`

#### API Functions

| Function | Type | Purpose |
|----------|------|---------|
| `comments.getComments(listingId)` | query | Fetch comments with scores, nested replies, sorted by score |
| `comments.getLatestComments()` | query | 50 most recent top-level comments with listing info |
| `comments.postComment(...)` | mutation | Create comment/reply with validation |
| `comments.seedComment(...)` | mutation | Test data seeding (custom timestamp) |
| `votes.vote(commentId, voterId, direction)` | mutation | Upsert vote: create/toggle/switch |
| `listings.upsertListing(...)` | mutation | Store listing metadata |

No cron jobs, no HTTP routes, no scheduled functions, no authentication middleware.

#### Backend Tests

- `tests/comments.test.ts` ‚Äî empty state, score sorting, validation (empty/length), reply nesting rules
- `tests/votes.test.ts` ‚Äî create vote, toggle off, switch direction, multi-voter scoring

### Build Notes

```bash
npm run build    # tsc -b && vite build && rm -f convex/*.js src/**/*.js tests/**/*.js
npm run dev      # vite (HMR dev server)
npm run test     # vitest
```

**Important**: `tsc -b` emits `.js` files alongside `.ts` sources. These `.js` files shadow the `.ts` files for the Convex bundler and vitest, causing "unexpected field" or "two output files" errors. Always run `rm -f convex/*.js` after `npm run build`. These files are in `.gitignore`.

## Key Technical Decisions

1. **Shadow DOM for style isolation** ‚Äî The extension UI is completely isolated from the host page's CSS. This prevents style conflicts and ensures the extension looks correct regardless of what polovniautomobili.com does with their styles.

2. **Convex for the backend** ‚Äî Real-time database with built-in React hooks (`useQuery`, `useMutation`). No need to write REST APIs or manage WebSocket connections. Comments and votes update in real time across all users.

3. **No authentication** ‚Äî Users are anonymous by design. The `anonymousId` (UUID) serves as the sole identity. This minimizes friction (no sign-up) but means identity is tied to the browser/device.

4. **Content script only (no background script)** ‚Äî Keeps the extension lightweight. The Convex client connects directly from the content script.

5. **Score-based sorting over chronological** ‚Äî Comments are ranked by community votes, surfacing the most useful content. Replies within a thread are also sorted by score.

6. **Single-level threading** ‚Äî Only one level of nesting (comment ‚Üí reply) to keep discussions focused and the UI simple. Reddit-style deep threading is intentionally avoided.

7. **Vote calculation at query time** ‚Äî Scores are computed dynamically from the votes table rather than stored as a denormalized field. Simpler data model, no sync issues, acceptable at current scale.

8. **Car-brand usernames** ‚Äî Auto-generated usernames use car brands (Golf, Punto, Astra, etc.) to fit the automotive domain. Creates a thematic identity without requiring user input.

9. **CRXJS for extension bundling** ‚Äî Vite plugin that handles Manifest V3 bundling, hot reload during development, and content script injection. Standard Vite dev experience for Chrome extension development.

10. **Listing metadata caching** ‚Äî When a user comments, the extension stores the listing's title/price/image/URL in the `listings` table. This allows the "Latest" feed to show listing cards without scraping the target site.
